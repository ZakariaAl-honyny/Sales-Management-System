# Implementation Plan: Products Module — Remaining Work

**Branch**: `028-products-module-complete` | **Date**: 2026-06-10 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/028-products-module-complete/spec.md`

## Summary

Complete the Products Module by implementing: (1) opening balance on product creation with automatic InventoryBatch("OPENING") and journal entry, (2) removing `Product.Cost` and deriving cost from InventoryBatches via FIFO, (3) renaming `HasExpiry` to `TrackExpiry` across all layers, (4) removing the deprecated `UnitBarcodes`/`ProductBarcode` table, and (5) adding Excel import for bulk product creation with opening stock support.

## Technical Context

**Language/Version**: C# .NET 10 LTS  
**Primary Dependencies**: ASP.NET Core 10, EF Core 10, FluentValidation 11, ClosedXML (already in project), QuestPDF (existing)  
**Storage**: SQL Server 2019+ via EF Core Fluent API  
**Testing**: xUnit + Moq + FluentAssertions (existing test infrastructure)  
**Target Platform**: Windows Desktop (WPF MVVM) + ASP.NET Core API (Windows Service)  
**Project Type**: Clean Architecture (6 projects: Domain, Contracts, Application, Infrastructure, Api, DesktopPWF)  
**Performance Goals**: Product creation < 2 seconds; Excel import of 100 products < 30 seconds  
**Constraints**: Must follow existing patterns — `Result<T>` in all services, `INotifyDataErrorInfo` in ViewModels, `ExecuteAsync()` wrapper for async commands, Fluent API only (no DataAnnotations), `DeleteBehavior.Restrict` on all FKs  
**Scale/Scope**: Small retail shops, single database, ~10k products max

## Phase 0: Research & Design Decisions

### Decisions to be made (all resolvable from existing codebase analysis):

1. **Opening batch creation flow**: Where exactly in `ProductService.CreateAsync()` does the batch get created — before or after `SaveChangesAsync`?
2. **Journal entry for opening balance**: Which service to use — `AccountingIntegrationService` or direct `_uow.JournalEntries.AddAsync`?
3. **Default warehouse for opening stock**: How to resolve the default warehouse ID?
4. **Product.Cost removal impact**: Which services read `Product.Cost` today that need migration to batch-based cost?
5. **FIFO cost query**: Where to implement — `IInventoryBatchRepository` or `IProductRepository`?
6. **Excel import format**: What columns, how to map categories?
7. **HasExpiry rename scope**: All 7 layers (Domain, Contracts, Application, Infrastructure, API, Desktop, Tests)
