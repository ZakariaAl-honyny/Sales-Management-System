## Summary

Replace all 13 placeholder `UserControl`s in `SalesSystem.Desktop` with fully functional WinForms modules that communicate with the ASP.NET Core API via `HttpClient`. Each module implements the Standard Module Pattern: a searchable/filterable list (`DataGridView` + `SearchBarControl`), an editor dialog/form, EventBus-driven cross-module refresh, role-based UI gating, and Arabic RTL layout. Invoice modules (Sales, Purchases, Returns, Transfers) include real-time financial calculation with a Tax Inclusive/Exclusive toggle. The Reports module exports to Excel (.xlsx) via ClosedXML and CSV via `StreamWriter`.

## Key Design Decisions

### Decision 1: Standard Module Pattern

Every CRUD module follows this exact structure (established from Products module):

```
[Entity]ListControl (UserControl)
  ├── ToolStripPanel: [Add] [Edit] [Deactivate/Reactivate] [Refresh]
  ├── SearchBarControl (debounced, 300ms)
  ├── CheckBox: "Show Deactivated" toggle
  └── DataGridView (ReadOnly, FullRowSelect, Arabic RTL)

[Entity]EditorForm (Form, Modal)
  ├── Title: "إضافة [entity]" / "تعديل [entity]"
  ├── Fields (RightToLeft=Yes, Arabic labels)
  ├── [Save] [Cancel] buttons
  └── Validation: non-empty + format checks
```

### Decision 2: Invoice Module Pattern

Invoice modules differ from CRUD modules — they use a single-form pattern:

```
[Invoice]ListControl (UserControl)
  ├── ToolStripPanel: [New] [View] [Post] [Cancel] [Refresh]
  ├── Date range pickers + Status filter
  └── DataGridView (summary per invoice)

[Invoice]Form (Form, Modal or Full-Screen)
  ├── Header: Customer/Supplier, Warehouse, Date, PaymentType
  ├── Tax Toggle: [☐ Tax Inclusive] + TaxRate field
  ├── Line Items Grid: [Product search] [Qty] [Price] [Discount] [LineTotal]
  ├── Totals Panel: SubTotal | TaxAmount | Discount | TotalAmount | Paid | Due
  ├── Status badge (Draft / Posted / Cancelled)
  └── Action Buttons: [Save Draft] [Post Invoice] [Cancel Invoice]
```

### Decision 3: Real-Time Calculation

Line item totals update on `CellEndEdit` of the DataGridView. The desktop computes **display-only** previews using the same formulas as the Domain (for UX responsiveness). The **authoritative** calculation always runs server-side on Post. The domain formula used for preview:

```csharp
// Display-only preview — Domain recomputes on save
decimal lineTotal = (qty * unitPrice) - discountAmount;
decimal subTotal  = items.Sum(i => i.LineTotal);
decimal taxAmount = isTaxInclusive
    ? subTotal - (subTotal / (1 + taxRate / 100m))
    : subTotal * (taxRate / 100m);
decimal totalAmount = subTotal - invoiceDiscount +
                      (isTaxInclusive ? 0 : taxAmount);
decimal dueAmount   = totalAmount - paidAmount;
```

> Note: This preview is purely for UX. The authoritative value is always returned from the API response and displayed after save/post.

### Decision 4: ClosedXML for Excel Export

```csharp
// ReportsControl.cs — Export to Excel
private void ExportToExcel(DataTable data, string reportName)
{
    using var workbook = new XLWorkbook();
    var ws = workbook.Worksheets.Add(reportName);
    ws.Cell(1, 1).InsertTable(data);
    
    using var dialog = new SaveFileDialog
    {
        Filter = "Excel Files|*.xlsx",
        FileName = $"{reportName}_{DateTime.Now:yyyyMMdd}.xlsx"
    };
    if (dialog.ShowDialog() == DialogResult.OK)
        workbook.SaveAs(dialog.FileName);
}

// Export to CSV
private void ExportToCsv(DataTable data, string reportName)
{
    using var dialog = new SaveFileDialog
    {
        Filter = "CSV Files|*.csv",
        FileName = $"{reportName}_{DateTime.Now:yyyyMMdd}.csv"
    };
    if (dialog.ShowDialog() != DialogResult.OK) return;
    
    using var writer = new StreamWriter(dialog.FileName, false, Encoding.UTF8);
    writer.WriteLine(string.Join(",", data.Columns.Cast<DataColumn>()
        .Select(c => $"\"{c.ColumnName}\"")));
    foreach (DataRow row in data.Rows)
        writer.WriteLine(string.Join(",", row.ItemArray
            .Select(v => $"\"{v}\"")));
}
```
