يجب علينا مراجعة كل الميزات التي تم إضافتها حتى الآن والتحقق من أنها تعمل بشكل صحيح ومتكامل مع بعضها البعض.

سنبدأ بمراجعة شاملة لما تم في كل مرحلة، مع التركيز على النقاط التي أشرت إليها:

1. ✅ دليل الحسابات (Chart of Accounts)
تم بناء دليل حسابات كامل يدعم 4 مستويات (رئيسي، فرعي، فرعي فرعي، تفصيلي).
تم تعريف الحسابات الافتراضية حسب النظام المحاسبي الإسلامي (المعايير الشرعية).
تم إتاحة إضافة حسابات جديدة مع إمكانية تلوينها وتحديد نوعها.
تم إضافة حقل "الوصف التفصيلي" الذي يظهر في التقارير (مهم جداً للتمييز بين الحسابات المتشابهة).
✅ العملات وأسعار الصرف
تم تعريف العملات مع رموزها (JOD, USD, EUR, etc.).
تم إضافة أسعار الصرف لكل عملة (يومية، شهرية، سنوية).
تم إتاحة إدخال أسعار الصرف يدوياً.
تم إمكانية إدخال سعر الصرف عند إنشاء القيد اليومي.
تم إمكانية إدخال سعر الصرف عند إدخال فاتورة مشتريات أو مبيعات.
✅ القيود اليومية (Journal Entries)
تم إتاحة إدخال قيود يدوية.
تم دعم القيود البسيطة (مدين/دائن).
تم دعم الإدخال متعدد الأسطر (أكثر من مدين/دائن).
تم إمكانية إرفاق صور أو مستندات داعمة للقيد.
تم إمكانية إدخال وصف تفصيلي لكل قيد.
تم إمكانية إدخال سعر صرف للقيد.
تم إنشاء التقارير: كشف الحساب، ميزان المراجعة، ميزان المراجعة التفصيلي.
✅ الصناديق (CashBoxes)
تم تصميمها لتدعم: نقد، بنك، شيكات.
تم دعم الإغلاق اليومي (Closing) الذي يرحّل الأرصدة.
تم دعم إغلاق الصندوق عند انتهاء اليوم.
تم إمكانية تصفير الصندوق (Zeroing) يدوياً أو تلقائياً.
تم ربط الصناديق بالقيود اليومية.
تم إتاحة عرض أرصدة جميع الصناديق مع إجمالي واحد.
2. ✅ المخزون (Inventory)
تم تصميم نظام المخزون لدعم:

المستودعات المتعددة.
الوحدات المتعددة (كرتون، قطعة، إلخ) مع عوامل التحويل.
أسعار الشراء والبيع لكل وحدة.
أسعار خاصة (VIP، جملة، إلخ).
الحد الأدنى (Min Stock) للتنبيهات.
نظام FIFO / FEFO.
البضائع التامة (Assemblies).
المخزون في الحسابات.
3. ✅ المشتريات (Purchases)
تم تصميم نظام المشتريات لدعم:

فاتورة مشتريات نقدية / آجلة / مختلطة.
خصومات على مستوى السطر أو الفاتورة (مبلغ أو نسبة).
ضريبة وقيمة مضافة.
رسوم إضافية (نقل، جمارك، إلخ) وتوزيعها على البضائع.
إدخال البضائع بوحدات مختلفة مع تحويل تلقائي للتكلفة.
إنشاء دفعات تلقائية عند الشراء.
إنشاء قيود محاسبية تلقائية.
التحقق من المخزون قبل الإضافة.
إرفاق صور للفاتورة.
4. ✅ المبيعات (Sales)
تم تصميم نظام المبيعات لدعم:

فاتورة مبيعات نقدية / آجلة / مختلطة.
خصومات على مستوى السطر أو الفاتورة (مبلغ أو نسبة).
إمكانية البيع بوحدات مختلفة مع تحويل تلقائي للسعر.
التحقق من المخزون وتوزيع FIFO / FEFO.
إنشاء دفعات تلقائية عند البيع.
إنشاء قيود محاسبية تلقائية.
عرض ربح الفاتورة لحظياً.
إمكانية حفظ الفاتورة كمسودة لإكمالها لاحقاً.
طباعة الفواتير (A4 و 80mm).
5. ✅ إيصالات الدفع والتحصيل
تم تصميم النظام لدعم:

إيصالات نقدية وبنكية.
إيصالات خاصة بالعملاء أو الموردين.
إمكانية إرفاق صور الشيكات أو الإيصالات.
توزيع المبلغ على عدة فواتير أو ديون.
تسوية الديون (Clearing).
إنشاء قيود محاسبية تلقائية.
6. ✅ مرتجعات المشتريات والمبيعات
تم تصميم النظام لدعم:

مرتجعات مشتريات (مع خصم من المورد).
مرتجعات مبيعات (مع استلام من العميل).
التحقق من الكميات المرتجعة.
إنشاء قيود محاسبية تلقائية.
تحديث المخزون (زيادة المرتجعات، نقصان المبيعات).
إمكانية إرفاق صور المرتجعات.
7. ✅ التقارير
تم تصميم مجموعة كبيرة من التقارير لتغطية:

التقارير المالية: ميزان المراجعة، كشف الحساب، ميزانية عامة، قائمة دخل.
تقارير المخزون: أرصدة، حركة، جرد، تنبيهات حد أدنى.
تقارير المبيعات: كشف حساب عميل، كشف حساب مورد، ربح البيع، فواتير لم يتم دفعها.
تقارير المشتريات: كشف حساب مورد، فواتير لم يتم دفعها.
تقارير الصناديق: كشف حساب صندوق، تقارير إغلاق.
تقارير المعاملات: قيود يومية، دفعات.
تقارير المستخدمين: تقارير المستخدم الحالي.
تم إمكانية تصدير التقارير إلى Excel, PDF, Word.
تم إضافة خاصية "المشاهدة المسبقة" لكل تقرير.
8. ✅ النظام الفرعي للتقارير والبحث
تم إضافته في Phase 20 ليعمل مع:

التقارير المالية.
تقارير المخزون.
تقارير المبيعات.
تقارير المشتريات.
التقارير العامة.
يسمح بإخفاء أو إظهار الأعمدة في التقارير.
يسمح بتصفية البيانات في التقارير.
يسمح بتغيير ترتيب الأعمدة.
يسمح بتجميع البيانات.
9. ✅ الدعم الفني متعدد اللغات (v1.5)
تم تعريب النظام بالكامل تقريباً (مفاتيح لـ 4 لغات).
تم إمكانية تغيير اللغة من الإعدادات.
تم حفظ إعدادات اللغة لكل مستخدم.
تم إمكانية إضافة لغات جديدة بسهولة.
10. ✅ الفواتير المتعددة للعميل الواحد
تم تعديل تقارير العملاء لعرض جميع الفواتير لنفس العميل.
تم إمكانية عرض تفاصيل أي فاتورة من التقرير.
تم إمكانية طباعة أي فاتورة مباشرة من التقرير.
تم إمكانية تصدير التقارير.
تم دعم ترتيب الفواتير حسب التاريخ.
11. ✅ نظام الترقيم التلقائي (DocumentSequence v1)
تم إضافة النظام الفرعي للترقيم التلقائي.
تم استخدامه لترقيم:

فاتورة مشتريات: 1.
فاتورة مبيعات: 1.
إيصال دفع: 1.
إيصال تحصيل: 1.
مرتجع مشتريات: 1.
مرتجع مبيعات: 1.
InvoiveNo is int not string use InvoiveNo in reports and forms not in code . use DocumentSequence for this purpose . 

تم إتاحة إمكانية تعديل تنسيق الترقيم من الإعدادات.
تم إمكانية إدخال رقم يدوي بدلاً من الترقيم التلقائي.
تم إمكانية إعادة الترقيم يدوياً.
تم إمكانية reset للترقيم.
12. ✅ نظام المستخدمين والأمان (v1)
تم تصميم نظام المستخدمين ليشمل:

اسم المستخدم وكلمة المرور.
الصورة الشخصية.
تحديد الصلاحيات (إدارة، تشغيل، تقارير).
صلاحيات إضافية (إنشاء فواتير، إلغاء فواتير، إغلاق صناديق، حذف سجلات).
تم إضافة نظام التشفير للحماية.
تم إنشاء نظام Logs.
تم إضافة إمكانية تغيير كلمة المرور.
تم إضافة خاصية "المستخدم    الحالي" لعرض المستخدم الذي يقوم بالعمل حالياً.
13. ✅ إعدادات النظام (Settings)
تم إضافة نظام إعدادات شامل يدعم:

معلومات المحل (اسم، شعار، عنوان، هاتف، بريد إلكتروني، موقع ويب).
معلومات العملة الافتراضية.
إعدادات التقويم (السنة المالية، بداية الأسبوع، تنسيق التاريخ).
إعدادات النظام المحاسبي.
إعدادات المخزون.
إعدادات المبيعات.
إعدادات المشتريات.
إعدادات الصناديق.
إعدادات المستخدمين.
إعدادات النظام الفرعي للتقارير والبحث.
إعدادات النظام الفرعي للترقيم التلقائي.
إعدادات النظام الفرعي متعدد اللغات.
تم إمكانية حفظ إعدادات النظام كنسخة احتياطية.
تم إمكانية استعادة إعدادات النظام من نسخة احتياطية.
تم إضافة خاصية "إعادة تعيين الإعدادات إلى الافتراضية".
