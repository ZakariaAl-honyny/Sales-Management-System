# أولاً: ما هو القيد اليومي؟

ببساطة شديدة:

القيد اليومي هو الترجمة المحاسبية لأي عملية تحدث داخل النظام.

مثال:

عندما تبيع منتجاً نقداً.

المستخدم يرى:

```text
تم بيع بطاطس بـ 1000 ريال.
```

لكن المحاسبة ترى:

```text
مدين:
الصندوق 1000

دائن:
المبيعات 1000
```

هذا يسمى قيداً يومياً.

-----------------------------------

مثال آخر

اشتريت بضاعة نقداً:

```text
5000 ريال
```

القيد:

```text
مدين:
المخزون 5000

دائن:
الصندوق 5000
```

-----------------------------------

مثال آخر

قبضت ديناً من عميل:

```text
3000 ريال
```

القيد:

```text
مدين:
الصندوق 3000

دائن:
العميل أحمد 3000
```

-----------------------------------

لذلك القيد اليومي ليس عملية مستقلة غالباً.

بل هو "الأثر المحاسبي" لأي عملية.

# 1. ما هو سند القبض؟

سند القبض يعني:

> دخول مال إلى الشركة أو المحل.

أمثلة:

```text

```

أو:

```text

```

أو:

```text
استلمت إيراداً آخر.
```

محاسبياً:

```text
مدين: الصندوق
دائن: العميل / الإيراد / رأس المال
```

---

# 2. ما هو سند الصرف؟

سند الصرف يعني:

> خروج مال من الشركة أو المحل.

أمثلة:

```text
دفعت إيجار المحل.
```

```text
سددت لمورد.
```

```text
دفعت مصاريف نقل.
```

محاسبياً:

```text
مدين: المصروف / المورد
دائن: الصندوق
```

---

# 3. هل المستخدم يحتاج يفهم مدين ودائن؟

لا.

في الواجهة يجب أن نجعلها سهلة.

بدلاً من أن نقول للمستخدم:

```text
اختر الحساب المدين
اختر الحساب الدائن
```

نقول له:

في سند القبض:

```text
استلمت من:
العميل / الحساب

إلى:
الصندوق
```

وفي سند الصرف:

```text
دفعت إلى:
المورد / المصروف / الحساب

من:
الصندوق
```

والنظام ينشئ القيد المحاسبي تلقائياً.


عند الضغط عليه يظهر:

```text
مدين:
الصندوق الرئيسي        10,000

دائن:
العميل أحمد            10,000
```

هذا يساعد المستخدم يتعلم بدون أن نفرض عليه المحاسبة.

---

# 8. الشرح داخل الشاشة

نفس الفكرة التي اتفقنا عليها.

بجانب عنوان الشاشة:

```text
سند قبض ⓘ
```

عند الضغط:

```text
سند القبض يستخدم عند دخول أموال إلى الصندوق أو البنك.

أمثلة:
• قبض من عميل
• إدخال رأس مال
• استلام إيرادات أخرى

يقوم النظام بإنشاء القيد المحاسبي تلقائياً.
```

وفي سند الصرف:

```text
سند الصرف يستخدم عند خروج أموال من الصندوق أو البنك.

أمثلة:
• دفع مصروف
• سداد مورد
• مسحوبات شخصية

يقوم النظام بإنشاء القيد المحاسبي تلقائياً.
```

---

# 9. هل نحتاج ربط السند بالفواتير؟

نعم، لكن نجعله اختيارياً.

مثلاً عند قبض من عميل:

يمكن أن يكون القبض:

## بدون ربط بفاتورة

```text
قبض عام من العميل
```

أو:

## مرتبط بفاتورة معينة

```text
قبض على فاتورة بيع رقم 15
```

أو مستقبلاً:

```text
قبض مبلغ واحد لتسديد عدة فواتير
```

لذلك نحتاج جدول ربط اختياري لاحقاً.

# 10. القواعد المهمة

## في سند القبض:

يجب أن يكون:

```text
المبلغ > 0
```

---

إذا العملة ليست العملة الأساسية:

```text
سعر الصرف مطلوب
```

---

إذا الحساب عميل:

```text
يتم تقليل مديونية العميل
```

---

إذا الحساب إيراد:

```text
يزيد الإيراد
```

---

## في سند الصرف:

إذا الصندوق لا يسمح بالسالب:

```text
لا يسمح بصرف مبلغ أكبر من رصيد الصندوق
```

مثال:

```text
رصيد الصندوق = 5,000
المبلغ المطلوب صرفه = 10,000
```

يظهر تنبيه:

```text
لا يوجد رصيد كافٍ في الصندوق.
```

# 11. حالة السند

أنصح أن يكون للسند حالة:

```text
مسودة
مرحل
ملغي
```

لكن لتسهيل النسخة الأولى:

زر الحفظ الأساسي يكون:

```text
حفظ وترحيل
```

يعني عند الحفظ:

- يتم حفظ السند.
- يتم إنشاء القيد اليومي.
- يتم تحديث حركة الصندوق.

ولا يتم تعديل السند بعد الترحيل.

إذا حصل خطأ:

```text
يتم إلغاء السند بعملية عكسية
```

وليس تعديله مباشرة.

# 14. هل CurrentBalance ضروري؟

يمكن وجوده كقيمة سريعة للعرض.

لكن محاسبياً الرصيد الحقيقي يجب أن يأتي من:

```text
حركة الحسابات / القيود اليومية
```

أو:

```text
CashTransactions
```

إذا قررنا الاحتفاظ بجدول حركة الصندوق.

الأفضل في V1:

نحتفظ بـ:

```text
CashTransactions
```

لتسريع عرض حركة الصندوق، لكن القيد المحاسبي هو المصدر النهائي للحقيقة.
